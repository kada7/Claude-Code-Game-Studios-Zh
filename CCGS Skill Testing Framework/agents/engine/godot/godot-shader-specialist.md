# Agent 测试规范：godot-shader-specialist

## Agent 概述
领域：Godot 着色语言（GLSL 衍生版本）、可视化着色器（VisualShader 图形）、材质设置、粒子着色器和后期处理效果。
不负责：游戏玩法代码、美术风格指导。
模型层级：Sonnet（默认）。
未分配 gate ID。

## 测试用例

### 用例 1：领域内请求 — 适当的输出
**输入：** "为 Godot 中的敌人死亡效果编写一个溶解效果着色器。"
**预期行为：**
- 生成有效的 Godot 着色语言代码（非 HLSL，非直接 GLSL）
- 使用 `shader_type spatial;` 或 `canvas_item` 作为适当选择
- 定义 `uniform float dissolve_amount : hint_range(0.0, 1.0);`
- 采样噪声纹理以确定逐像素溶解阈值
- 对低于阈值的像素使用 `discard;`
- 可选地在溶解边界附近使用 emission 添加边缘发光效果
- 代码语法符合 Godot 着色语言规范

### 用例 2：HLSL 重定向
**输入：** "为此溶解效果编写一个 HLSL 计算着色器。"
**预期行为：**
- 不生成 HLSL 代码
- 明确声明："Godot 不直接使用 HLSL；它使用自己的着色语言（GLSL 衍生版本）"
- 将 HLSL 意图转换为等效的 Godot 着色器方法
- 注意 RenderingDevice 计算着色器在 Godot 4 中可用，但属于低级 API，如果这是意图则适当标记

### 用例 3：截止日期后的 API 变更 — 纹理采样（Godot 4.4）
**输入：** "在着色器中使用 `texture()` 和 sampler2D 采样噪声纹理。"
**预期行为：**
- 检查版本参考：Godot 4.4 更改了纹理采样器类型声明
- 标记潜在的 API 变更：`sampler2D` 语法和 `texture()` 调用行为可能与 4.4 之前版本不同
- 根据项目固定版本（4.6）提供正确的语法，如迁移说明中所述
- 不在未标记版本风险的情况下使用 4.4 之前的纹理采样语法

### 用例 4：片段着色器 LOD 策略
**输入：** "水面片段着色器有 8 个纹理采样，导致中端硬件上的 GPU 瓶颈。"
**预期行为：**
- 识别逐片段纹理采样计数为主要成本驱动因素
- 提出 LOD 策略：
  - 在远处减少采样计数（基于距离的着色器变体或 LOD 级别）
  - 离线预烘焙一些纹理组合
  - 对远处采样使用较低分辨率的噪声纹理
- 提供实现 LOD 方法的着色器代码修改
- 不改变水面系统的游戏玩法行为

### 用例 5：上下文传递 — Godot 4.6 发光效果重做
**输入：** 引擎版本上下文：Godot 4.6。请求："为场景添加 bloom/glow 后期处理效果。"
**预期行为：**
- 引用 VERSION.md 说明：Godot 4.6 包含发光效果重做
- 使用 4.6 WorldEnvironment 方法生成发光配置指导，而非 4.6 之前的 API
- 明确说明 4.6 发光效果重做中哪些属性或参数发生了变化
- 标记由于截止日期后时间安排，LLM 训练数据可能包含错误信息的任何属性
