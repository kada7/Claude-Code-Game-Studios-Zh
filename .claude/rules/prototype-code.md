# 原型代码标准（宽松）

原型是用于验证想法的一次性代码。标准有意放宽，以最大化迭代速度。目标是学习，而不是生产质量。

## 在原型中允许的内容
- 硬编码数值（无需数据驱动配置）
- 最少或无文档注释
- 简单架构（无需依赖注入）
- 单例和全局状态
- 复制粘贴的代码（无需抽象）
- 保留调试输出
- 占位美术和音频
- 快速而临时的解决方案

## 仍然需要的要求
- 每个原型位于独立的子目录中：`prototypes/[name]/`
- 每个原型**必须**有一个 `README.md`，包含：
  - 正在测试的假设
  - 如何运行原型
  - 当前状态（进行中 / 已结束）
  - 发现结果（原型结束时更新）
- 生产代码**不得**引用或导入 `prototypes/` 中的内容
- 原型不能修改 `prototypes/` 之外的文件
- 原型不能被部署或发布

## 当原型成功时
如果一个原型验证了一个概念，并且该功能进入生产阶段：
1. 原型代码**不**直接迁移 — 按生产标准重新编写
2. 原型 `README.md` 的发现结果将指导生产设计文档的编写
3. 原型目录保留以供参考，但不再扩展

## 清理
已结束的原型应在发现结果被捕获后进行归档或删除。
切勿让原型代码通过逐步“清理”演变成生产代码。
