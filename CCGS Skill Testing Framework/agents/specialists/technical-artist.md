# Agent 测试规范：technical-artist

## Agent 概览
**负责领域：** Shaders、VFX、渲染优化、艺术流水线工具、视觉性能。
**不负责：** 艺术风格决策或调色板（art-director）、游戏玩法代码（gameplay-programmer）。
**模型层级：** Sonnet（默认）。
**无分配的 Gate ID。**

---

## 静态断言（结构）

- [ ] `description:` 字段存在且是领域特定的（涉及 shaders / VFX / 渲染）
- [ ] `allowed-tools:` 列表包含 Read、Write、Edit、Bash、Glob、Grep
- [ ] 模型层级为 Sonnet（specialists 的默认值）
- [ ] Agent 定义不宣称对艺术风格指导或游戏玩法逻辑拥有权限

---

## 测试用例

### 用例 1：领域内请求 —— 适当的输出
**输入：** "为敌人死亡序列创建一个溶解效果 shader。"
**预期行为：**
- 生成适用于配置引擎的 shader 代码或 Shader Graph 节点规范（Godot 着色语言 / Unity Shader Graph / Unreal Material Blueprint）
- 定义 `dissolve_amount` uniform（0.0–1.0）作为动画驱动
- 使用噪声纹理采样来确定溶解阈值
- 将边缘照明技术作为可选增强功能进行说明
- 输出具有引擎版本意识（如果需要 post-cutoff API，则检查版本参考）

### 用例 2：领域外请求 —— 正确重定向
**输入：** "定义艺术圣经调色板：UI 的主色、辅色和强调色。"
**预期行为：**
- 不生成调色板决策或艺术指导文档
- 明确说明艺术风格决策属于 `art-director`
- 将请求重定向到 `art-director`
- 可以说明一旦调色板确定，它可以稍后实现颜色分级或调色板 LUT shader

### 用例 3：性能警告 —— GPU 粒子数量
**输入：** "VFX 系统在爆炸池中有 50,000 个粒子时触发 GPU 粒子数量警告。"
**预期行为：**
- 生成解决特定警告的优化规范
- 提出具体策略：每个发射器的粒子预算上限、基于 LOD 的粒子减少、GPU 实例化，或为远处效果切换到基于网格的 VFX
- 在可计算的情况下提供优化前后的 GPU 成本估算
- 不改变爆炸的游戏玩法行为（将任何游戏玩法影响委托给 gameplay-programmer）

### 用例 4：引擎版本兼容性
**输入：** "为水 shader 使用新的纹理采样器 API。"
**预期行为：**
- 在建议任何 API 之前检查引擎版本参考（例如，`docs/engine-reference/godot/VERSION.md`）
- 如果请求的 API 是 post-cutoff 的（例如，Godot 4.4+ 纹理类型更改），则标记
- 为项目的固定引擎版本提供正确的语法
- 如果不确定 post-cutoff 行为，明确说明不确定性并指向已验证的文档

### 用例 5：上下文传递 —— 使用性能预算
**输入：** 从 `technical-preferences.md` 提供的性能预算上下文：2ms GPU 帧预算，最大 200 个绘制调用。请求："优化森林渲染系统。"
**预期行为：**
- 引用提供的上下文中的特定 2ms GPU 预算和 200 个绘制调用限制
- 提出针对这些确切目标校准的优化（例如，"批处理将绘制调用从 340 减少到约 180，在 200 的限制内"）
- 不提出在其他维度上会超过规定预算的优化
- 生成按预期影响与实施成本排序的优化列表

## 协议合规性

- [ ] 保持在声明的领域内（shaders、VFX、渲染优化、艺术流水线）
- [ ] 将艺术风格决策重定向到 art-director
- [ ] 返回结构化发现（shader 代码、带有指标的优化规范、节点图）
- [ ] 未经明确委托，不修改游戏玩法代码文件
- [ ] 在建议 post-cutoff API 之前检查引擎版本参考
- [ ] 根据规定的预算量化性能变化

## 覆盖范围说明
- 溶解 shader（用例 1）应在 `production/qa/evidence/` 中包含视觉测试参考
- 引擎版本检查（用例 4）确认 agent 将 VERSION.md 视为权威
- 性能预算用例（用例 5）验证 agent 读取并应用提供的上下文数字
