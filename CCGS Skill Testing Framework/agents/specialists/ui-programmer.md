# Agent Test Spec: ui-programmer

## Agent 概述
领域：菜单屏幕、HUD、库存屏幕、对话框、UI 框架代码和数据绑定。
不负责：UX 流程设计（ux-designer）、视觉风格指导（art-director / technical-artist）。
模型层级：Sonnet（默认）。
未分配门禁 ID。

---

## 静态断言（结构）

- [ ] `description:` 字段存在且具有领域特定性（引用菜单/HUD/UI 框架/数据绑定）
- [ ] `allowed-tools:` 列表包含 Read、Write、Edit、Bash、Glob、Grep
- [ ] 模型层级为 Sonnet（专家默认）
- [ ] Agent 定义不声称对 UX 流程设计或视觉艺术指导拥有权限

---

## 测试用例

### 用例 1：领域内请求 — 适当的输出
**输入：** "根据 `design/ux/inventory-flow.md` 中的 UX 规范实现库存屏幕。"
**预期行为：**
- 在生成任何代码之前读取 UX 规范
- 使用项目配置的 UI 框架（UI Toolkit、UGUI、UMG 或 Godot Control 节点）生成实现
- 实现规范中定义的所有状态（默认、悬停、选中、空槽、锁定槽）
- 通过项目的数据模型将库存数据绑定到 UI 元素，而非硬编码值
- 根据编码标准在公共 UI API 上包含文档注释

### 用例 2：领域外请求 — 正确重定向
**输入：** "设计库存交互流程 — 当玩家装备、丢弃或组合物品时会发生什么。"
**预期行为：**
- 不生成交互流程设计或用户流程图
- 明确说明 UX 流程设计属于 `ux-designer`
- 将请求重定向到 `ux-designer`
- 注明一旦流程规范就绪，它可以实现该规范

### 用例 3：自定义动画协调
**输入：** "库存中的物品选中时需要自定义弹跳动画。"
**预期行为：**
- 认识到定义动画曲线和感觉属于 technical-artist 的领域
- 在没有规范的情况下不发明动画参数（时序、缓动）
- 与 `technical-artist` 协调获取动画规范（持续时间、缓动曲线、过冲量）
- 一旦提供规范，生成将动画绑定到选中状态的实现

### 用例 4：模糊的 UX 规范 — 标记返回
**输入：** UX 规范说明"选中时显示物品详情"，但未定义选中空槽时会发生什么。
**预期行为：**
- 识别规范中的模糊性（空槽选中状态未定义）
- 不为未定义状态做出任意的实现决策
- 将模糊性标记回 `ux-designer`，并附上具体问题："当选中空库存槽时，详情面板应显示什么？"
- 可以提出两个常见选项（隐藏面板/显示占位符）以帮助 ux-designer 快速决策

### 用例 5：上下文传递 — 引擎 UI 工具包
**输入：** 提供引擎上下文：项目使用带有 Control 节点 UI 的 Godot 4.6。请求："为库存实现可滚动物品列表。"
**预期行为：**
- 使用 Godot 的 `ScrollContainer` + `VBoxContainer` + `ItemList`（或等效）模式，而非 Canvas 或 UGUI
- 不为 Godot 项目生成 Unity UGUI 或 Unreal UMG 代码
- 在使用特定 API 之前检查引擎版本参考（4.6）是否有任何 Control 节点 API 从 4.4/4.5 发生更改
- 生成与项目配置语言一致的 GDScript 或 C# 代码

## 协议合规性

- [ ] 保持在声明的领域内（菜单、HUD、UI 框架、数据绑定）
- [ ] 将 UX 流程设计重定向到 ux-designer
- [ ] 在实现动画之前与 technical-artist 协调获取动画规范
- [ ] 将模糊的 UX 规范标记回 ux-designer，而非做出任意的实现决策
- [ ] 返回结构化输出（实现代码、数据绑定模式、UI 状态的状态机）
- [ ] 为项目使用正确的引擎 UI 工具包 — 绝不使用跨引擎代码

---

## 覆盖范围说明
- 库存实现（用例 1）应在 `production/qa/evidence/` 中包含 UI 交互测试或手动演练文档
- 动画协调（用例 3）确认 Agent 在没有规范的情况下不发明感觉参数
- 模糊规范（用例 4）验证 Agent 将规范缺口路由回创作 Agent 而非猜测
